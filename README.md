ledfx-trigger
=============

Now at 0.4.4!
-------------

![Build Status](https://github.com/armyofevilrobots/ledfx-trigger/actions/workflows/rust.yml/badge.svg)

![](web/doppler-wall-animated.webp)

LedFx-Trigger provides a `--user` system service which monitors your audio devices,
which will look for output on a configured audio device, and automatically
pause/unpause LEDFX depending on whether there is something playing. 

LedFx-Trigger can be installed via `cargo install --path .`

The configuration file will be autogenerated at startup if it does not yet exist.
Below is a simple guide to the settings:

```ron
(
    loglevel: 3,  // 0: no logging, 1: error, 2: warn, 3: info, 4: debug, 5: TRACE
    logfile: Some("/home/yourname/.ledfx-trigger/ledfx-trigger.log"),  // Where to log
    audio_config: Some(AudioConfig(  // Optional audioconfig for monitoring.
            input_device: "default", //iec958:CARD=J380,DEV=0",
            jack: false,  // I've not tested jack integration. YMMV.
            ledfx_threshold_db: Some(-32.),  // How many db minimum to keep vis on.
        )),
    ledfx_url: Some("http://localhost:8888"), // If set to None, ledfx won't be modified.
    ledfx_idle_cycles: Some(5), // How many $CYCLE_SECONDS second cycles of silence before pausing ledfx 
    cycle_seconds: 10.0, // How many seconds between updates. Default 10.0 seconds
    restart_on_cfg_change: false // true -> Great if you're using systemd
)
```

SystemD
-------

To install as a SystemD service:

```bash
cp systemd/ledfx-trigger.service ~/.config/systemd/user/
systemctl --user enable ledfx-trigger
systemctl --user start ledfx-trigger
```

*Note:* There is some nominal multi-platform support in this codebase, but it has not been
tested outside of Linux (specifically Ubuntu/PopOS 22.04 and Arch). I made this thing to
scratch my own itch, but it might be useful for other people with a little elbow grease.

There is a file named `dependencies.arch.txt` which contains a list of packages required
for this program to work. Most are installed by default, but this makes it explicit if you
have any linker errors on startup. `pacman -Su --needed $(cat dependencies.arch.txt)` should do
the trick for you in that case.
